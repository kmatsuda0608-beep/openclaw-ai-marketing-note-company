# Ideal Sheet Structure v1

Last updated: 2026-04-17
Brand: `nanda AI`

## 結論
スプレッドシートは **1ファイル内でタブを分ける** のが最適。

## 推奨タブ構成

### 1. `管理`
全体の進行管理用。

最低限ほしい列:
- 日付
- テーマ
- 情報ソース
- 優先度
- 使用先 (X / Threads / 両方)
- ステータス
- 投稿URL
- 振り返り分析
- 振り返りFB
- 再学習済み

### 2. `X投稿`
X専用の本文管理。

最低限ほしい列:
- 日付
- リサーチ情報
- X数値
- X投稿文
- 共通ベーステキスト
- ハッシュタグ
- ステータス
- 修正案
- AI更新
- X投稿URL
- 振り返り分析
- 振り返りFB
- 再学習済み

### 3. `Threads投稿`
Threads専用の本文管理。

最低限ほしい列:
- 日付
- リサーチ情報
- Threads数値
- Threads投稿文
- 共通ベーステキスト
- ハッシュタグ
- ステータス
- 修正案
- AI更新
- Threads投稿URL
- 振り返り分析
- 振り返りFB
- 再学習済み

## いまのシート運用
今すぐ別タブへ分けなくても運用はできる。
ただし今後、投稿本数が増えるほど X と Threads を同じタブで持つと見づらくなる。

なのでおすすめは以下。
1. いまは現在タブで運用継続
2. 本数が10〜20本たまったら `X投稿` と `Threads投稿` を分離
3. `管理` タブを後から追加

## 運用ルール
- 1行目はヘッダ固定
- 実データは必ず2行目から記載
- 投稿前は数値列を空欄でよい
- 投稿後に実数値とURLを追記
- FBを理解したら `再学習済み = 済`
- FBが曖昧なら質問してから `済` にする

## 現時点のおすすめ
現時点では、**ファイルは1つ、タブは将来的に3つへ分離** がベスト。
