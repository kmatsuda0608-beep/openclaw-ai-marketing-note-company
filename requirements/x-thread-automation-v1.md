# X Thread Automation v1

Last updated: 2026-04-16
Brand: `nanda AI`

## 1. 方針

全SNSを同時に自動化するのではなく、**まずはXスレッドの自動投稿だけに絞る**。
これは負荷分散のためだけでなく、運用と改善の単位を小さくするためでもある。

## 2. 目的

- 自動化対象を1チャネルに絞って失敗コストを下げる
- スレッド形式で、単発ポストより情報量のある発信を安定化する
- 投稿管理、承認、投稿後ログを一元管理できるようにする

## 3. 初期運用の考え方

### Phase 1
- スプレッドシートで投稿管理
- シエルがスレッド案を作成
- 人が確認
- 自動投稿 or 半自動投稿
- 投稿URLと結果をログへ戻す

### Phase 2
- 投稿結果を見てフックや構成を改善
- 反応のよい型をテンプレ化
- CTAの種類を増やす

### Phase 3
- 必要なら無料noteや他SNSへ展開
- ただしXで勝ち筋が見えるまでは横展開しない

## 4. スプレッドシート運用について

**結論: スプレッドシート管理で問題ない。むしろ推奨。**

理由:
- 投稿ネタ、下書き、承認状況、投稿URLを一画面で追える
- おまつさん側でも確認しやすい
- 初期は仕様が変わりやすく、シートの方が調整が速い
- 将来CSVやAPI連携に移しやすい

## 5. シートに入れるべき主な列

- ID
- ステータス
- 優先度
- テーマ
- 狙い
- 想定読者
- フック
- ポスト1〜10
- CTA
- 参照リンク
- 承認状況
- 投稿日時
- 投稿URL
- 結果メモ

## 6. 初期ステータス案

- Idea
- Draft
- Review
- Approved
- Scheduled
- Posted
- Hold
- Rework

## 7. 自動投稿に入る前の前提条件

- Xアカウントが作成済みであること
- 投稿方法を決めること
  - X API
  - ブラウザ自動操作
  - 外部自動化ツール連携
- 投稿前の最終承認ルールがあること
- エラー時の差し戻し手順があること

## 8. 今のおすすめ

最初は以下で進めるのがよい。

1. スプレッドシートでネタとスレッド本文を管理
2. まず10本程度の下書きをためる
3. 反応を見ながら型を固定
4. その後に投稿処理を自動化
