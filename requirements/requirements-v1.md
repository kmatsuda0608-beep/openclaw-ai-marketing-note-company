# Requirements v1

Last updated: 2026-04-13
Project: `openclaw-ai-marketing-note-company`

## 1. Project overview

本プロジェクトは、参照したX投稿の“AI運用組織”の構造を参考にしつつ、**Claude Codeそのものを再現するのではなく、OpenClaw上で同等の運用構造を再現**することを目的とする。

対象テーマは **AI / Webマーケティング（広告）**。
主な販売導線は **X → 無料note → 有料note** とし、初期段階では人の確認を前提とした半自動運用を行う。

## 2. すでに確定している前提

- Claude Codeそのものではなく、**OpenClawで同等構造を作る**
- 初期MVPは**自動投稿なし**
- 公開フローは **下書き生成 → 人確認 → 手動公開**
- **5人部隊はできるだけそのまま再現**する
- 初期テーマは **AI / Webマーケ（広告）**
- ローカルプロジェクト名は **`openclaw-ai-marketing-note-company`**
- GitHubへの同期は後続。現時点ではローカル整備を先行する

## 3. ゴール

### 3.1 事業ゴール
- Xで認知を獲得する
- 無料noteで教育と信頼構築を行う
- 有料noteで収益化の導線を作る
- 日次で回せるコンテンツ運用体制を構築する

### 3.2 プロダクトゴール
- AI 5部隊が役割分担して成果物を出せる
- 人が短時間で確認・承認しやすい状態を作る
- 出力と判断履歴がログとして残る
- GitHubに載せやすい構成で成果物を整理する

## 4. 体制

### 4.1 5人部隊構成
1. **オーケストレーター**
   - 全体の指示整理
   - 依頼の分配
   - 当日のテーマ整理
   - 進行管理
2. **X集客部隊**
   - X投稿案の作成
   - 認知獲得用の短文案作成
   - プロフィール・無料noteへの導線設計
3. **無料note部隊**
   - 教育・信頼構築のための無料記事案作成
   - 見込み読者に価値を与える記事構成の作成
4. **有料note部隊**
   - 有料商品の企画・構成・販売文作成
   - 商品として成立する章立てと価値整理
5. **販売導線部隊**
   - CTA、販売導線、導線改善案の作成
   - X / 無料note / 有料note 間の遷移設計

### 4.2 人間の役割
- 最終判断者
- 公開可否の承認者
- ブランド・訴求・商材の責任者
- AI出力の事実確認者

## 5. 対象読者の初期定義

初期ターゲットは以下を想定する。

- AI活用に興味がある個人事業主
- 広告運用やWebマーケ実務に関わる人
- SNS運用やコンテンツ販売を仕組み化したい人
- 情報は欲しいが、煽り系・根拠の薄い発信は避けたい人

## 6. MVP定義

### 6.1 MVPの意味
本プロジェクトにおけるMVPとは、**最初から完全自動運用を目指さず、実際に運用可能で価値検証できる最小構成**を指す。

### 6.2 MVPで必ず入れるもの
- 5人部隊の役割定義
- 入力フォーマットの定義
- 出力フォーマットの定義
- X投稿案の生成
- 無料note案の生成
- 有料note案の生成
- CTA / 販売導線案の生成
- 承認フロー
- 実行ログ / 改善メモの保存
- GitHubに移しやすいフォルダ構成

### 6.3 MVPでは入れないもの
- Xへの完全自動投稿
- noteへの完全自動投稿
- 売上保証ロジック
- CRM / 決済 / CS / 会計の自動化
- 実績不明の数字を使った訴求自動化

## 7. 機能要件

### 7.1 オーケストレーター
- 当日のテーマ、素材、優先順位を整理できる
- 各部隊への依頼文を組み立てられる
- 部隊ごとの成果物を回収し、レビューキューへ載せられる
- 日次・週次の振り返り項目をまとめられる

### 7.2 X集客部隊
- 認知獲得向けのX投稿案を複数案出せる
- プロフィール遷移や無料note遷移を意識したCTA案を出せる
- 投稿タイプを分けて出力できる
  - 学び系
  - 気づき系
  - 実務ノウハウ系
  - 導線系

### 7.3 無料note部隊
- タイトル案
- リード文
- 見出し構成
- 本文たたき台
- 有料noteへの自然なCTA
を出力できる

### 7.4 有料note部隊
- 商品コンセプト
- 想定読者
- ベネフィット整理
- 章立て
- 販売ページ草案
- 注意書き / 期待値調整表現
を出力できる

### 7.5 販売導線部隊
- X → 無料note
- 無料note → 有料note
- 有料note → 次回商品 / 継続接点
の導線を整理できる
- CTA文面を複数案出せる
- 導線のボトルネック仮説を記録できる

### 7.6 承認フロー
- 生成物はすべて公開前に人が確認する
- ステータスは最低限以下を持つ
  - Draft
  - Review
  - Approved
  - Rework
  - Rejected
- 差し戻し理由をメモできる

### 7.7 ログ / 改善メモ
- 日付ごとに実行ログを残せる
- どの素材から何を出したか追跡できる
- 良かった表現、危険だった表現、改善メモを蓄積できる

## 8. 非機能要件

- 日本語で実務的にわかりやすく出力する
- 煽りすぎない
- 誇大表現を避ける
- 元素材にない事実を断定しない
- 景表法・各種規制・プラットフォーム規約に配慮する
- 匿名寄りブランド運用に耐えるトーンにする
- 後からGitHubで管理しやすい命名・配置にする

## 9. KPI要件

KPIの詳細値は `requirements/kpi-draft.md` を初期基準とする。

### 9.1 North Star候補
- 月間の**有料note販売ページ到達数**

### 9.2 初期KPI（30〜90日）
#### X
- 月間投稿数: 20〜30本
- 1投稿あたり平均インプレッション: 300〜1000
- エンゲージメント率: 0.5%以上目標
- プロフィール / リンク遷移率: 0.5〜1.5%

#### 無料note
- 月間公開 / 更新本数: 4〜8本
- 月間PV: 300〜1000
- 無料note → 有料note遷移率: 10〜20%

#### 有料note
- 販売ページCVR: 1〜3%
- 30日販売目標: 1〜3件
- 90日販売目標: 5〜15件

### 9.3 KPI運用方針
- 最初は売上最大化より**導線成立**を優先する
- 測るべき順番は以下
  1. 投稿が回るか
  2. 導線がつながるか
  3. CVRが測れるか
  4. 改善サイクルが回るか
