# Requirements v3 (Build Scope Draft)

Last updated: 2026-04-16
Project: `openclaw-ai-marketing-note-company`

## 1. 今回の依頼の整理

参照X投稿の構造をベースに、OpenClaw上で同等レベルの「AI運用組織」を再現する。
ただし、完全コピーではなく、**おまつさんの事業文脈に合わせて実運用可能な形に落とし込む**ことを前提とする。

今回の開始点は以下。
- まずは要件定義から始める
- GitHubはおまつさん側で用意中
- GitHub準備完了後、成果物をリポジトリに整理して集約する
- 今後いろんな作業を切り替えやすいよう、用途別にGitHub管理を分ける

## 2. 作るものの定義

作る対象は、以下を回すための OpenClaw ベース運用体制。

- Xでの認知獲得
- 無料noteでの教育・信頼構築
- 有料noteでの収益化
- 各工程の改善メモ、判断履歴、再利用素材の蓄積

つまり単発の投稿生成ツールではなく、**継続運用できる制作組織**を作る。

## 3. 再現したいコア構造

参照元から再現したい本質は以下。

1. 人が全部を抱えず、役割分担させる
2. 日々の制作を仕組みとして回す
3. X → 無料note → 有料note の導線を一つの流れで設計する
4. 人は最終判断と修正に集中する
5. AIの出力を蓄積し、次回以降の改善に活かす

## 4. 初期MVP要件

### 4.1 MVPで必須
- 5人部隊の役割定義
- 各部隊への入力テンプレート
- 各部隊の出力テンプレート
- 当日テーマから制作物を組み立てる運用フロー
- X投稿案の生成
- 無料note案の生成
- 有料note案の生成
- CTA / 販売導線案の生成
- レビュー / 承認ステータス管理
- 実行ログと改善メモの保存

### 4.2 MVPではまだやらない
- Xへの自動投稿
- noteへの自動投稿
- DMsや営業連絡の自動送信
- 売上や実績の自動誇張訴求
- 人の確認を飛ばした公開

## 5. 5人部隊の役割

### 5.1 オーケストレーター
- 当日のテーマ整理
- 素材回収
- 各部隊への依頼分配
- 出力回収
- レビューキュー整理

### 5.2 X集客部隊
- 認知獲得用ポスト案
- 興味喚起フック
- 無料noteへの導線設計
- 投稿タイプ別の複数案出し

### 5.3 無料note部隊
- 無料記事の企画
- タイトル案
- 構成案
- 本文たたき台
- 有料noteへの自然な接続

### 5.4 有料note部隊
- 商品企画
- ベネフィット整理
- 章立て
- 販売ページ草案
- 注意書きと期待値調整

### 5.5 販売導線部隊
- X → 無料note の導線改善
- 無料note → 有料note のCTA改善
- ボトルネック仮説の記録
- 導線全体の見直し提案

## 6. 人の役割

人が担う責任は以下。

- テーマの最終決定
- ブランド方針の決定
- 事実確認
- 公開可否判断
- AI出力の修正指示

初期段階では、**AIは制作補助と構造化、最終公開は人** という分担に固定する。

## 7. 制作対象の初期テーマ

現時点の初期テーマは以下。

- AI
- Webマーケティング
- 広告運用

初期ターゲットは以下を想定する。
- AI活用に興味がある個人事業主
- 集客や販売導線を仕組み化したい人
- SNS発信を事業に繋げたい人
- 実務寄りの情報を求める人

## 8. 運用フロー要件

1. 素材収集
2. 当日テーマ確定
3. オーケストレーターが依頼文作成
4. 各部隊が成果物生成
5. 人がレビュー
6. 修正 or 承認
7. 公開
8. ログ保存
9. 次回改善点を記録

## 9. 品質・安全要件

- 日本語で実務的にわかりやすいこと
- 煽りすぎないこと
- 実績不明の数字を断定しないこと
- 景表法、広告表現、プラットフォーム規約に配慮すること
- 匿名寄りブランド運用に耐えること
- ローカル秘密情報をGitHubに含めないこと

## 10. GitHub運用要件

### 10.1 基本方針
GitHubは「接続できること」より、**用途別に切り分けて継続運用できること** を優先する。

### 10.2 この案件の扱い
本案件は `openclaw-ai-marketing-note-company` として独立管理する。
この中に以下を集約する。
- 要件定義
- プロンプト
- runbook
- workflow
- テンプレ
- ログ雛形

### 10.3 用途別に分ける意図
- 案件ごとに文脈を汚さない
- 目的ごとに履歴を追いやすい
- 別案件へ切り替えやすい
- 共有範囲や公開範囲を調整しやすい

## 11. 成功条件

初期成功条件は以下。

- 5人部隊の構造が文書として定義されている
- 日次制作フローが回せる
- X / 無料note / 有料note の導線が一連で設計されている
- 人レビュー前提で実運用可能な出力が出せる
- GitHub準備完了後に整理して載せ替えできる
